# CLAUDE.md

이 파일은 Claude Code가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

마크다운(.md) 파일을 읽어서 정적 블로그 웹사이트로 변환하는 프로젝트.

- 글쓰기는 마크다운으로, 결과물은 HTML 블로그 페이지로.
- 프레임워크/빌드 툴 없이 순수 HTML, CSS, JavaScript로만 구현.
- 디자인 원칙: 깔끔하고 읽기 좋은 타이포그래피, 다크 모드 지원, 모바일 반응형.

## 기술 스택 원칙

- **No framework**: React, Vue, Next.js 등 사용하지 않음. 번들러(Webpack/Vite)도 사용하지 않음.
- **No build step 지향**: 브라우저에서 바로 동작하는 순수 JS(ES 모듈)만 사용. 트랜스파일 불필요.
- 마크다운 파싱은 의존성을 최소화. 외부 라이브러리를 쓸 경우 CDN 스크립트 태그 하나로 끝나는 수준만 허용하고, 가능하면 직접 구현.
- 정적 파일만으로 동작해야 함 (서버 사이드 렌더링/DB 없음). GitHub Pages 등 정적 호스팅으로 배포 가능해야 함.

## 디자인 가이드

- 타이포그래피 중심 레이아웃. 본문 가독성을 최우선으로 (적절한 줄 간격, 본문 최대 너비 제한 등).
- 다크 모드: `prefers-color-scheme` 미디어쿼리 기반 자동 감지 + 수동 토글 버튼 지원. CSS 변수(`:root`, `[data-theme="dark"]`)로 색상 관리.
- 모바일 우선 반응형: 미디어 쿼리로 작은 화면에서도 여백/폰트 크기 최적화.
- 불필요한 애니메이션이나 장식 요소 지양. 콘텐츠가 주인공.

## 작업 시 유의사항

- 새 의존성을 추가하기 전에 순수 JS로 구현 가능한지 먼저 검토할 것.
- 컴포넌트화가 필요하더라도 프레임워크 없이 순수 JS 함수/템플릿 리터럴로 구성.
- 파일 구조는 단순하게 유지 (예: `index.html`, `styles.css`, `main.js`, `posts/*.md`).
