Claude Code로 블로그를 만들면서 HTML, CSS, JavaScript가 각각 어떤 역할을 하는지 다시 한번 정리해봤습니다.

## HTML — 구조

페이지의 뼈대를 담당합니다. 이 블로그에서는 `index.html`이 글 목록을, `post.html`이 글 상세 페이지의 틀을 잡아줍니다. 실제 내용은 비어 있고, JavaScript가 채워 넣는 구조입니다.

## CSS — 스타일

색상, 여백, 폰트, 레이아웃을 담당합니다. 다크 모드는 CSS 변수(`--bg`, `--text` 등)를 `:root`와 `[data-theme="dark"]`에 각각 정의해두고, JavaScript가 `data-theme` 속성만 바꿔주는 방식으로 구현했습니다. 모바일 대응은 `@media` 쿼리로 화면 너비에 따라 여백과 글자 크기를 조정했습니다.

## JavaScript — 동작

가장 많은 역할을 합니다.

- `posts/index.json`을 불러와 글 목록을 렌더링
- 마크다운 파일을 fetch해서 직접 만든 파서로 HTML로 변환
- 검색어와 태그로 목록을 필터링
- 버튼 클릭에 따라 다크/라이트 모드를 전환하고 `localStorage`에 저장

프레임워크 없이 만들다 보니 평소 라이브러리가 대신 해주던 일들(마크다운 파싱, 상태 관리, 라우팅)을 직접 구현하게 되어서, 각 기술이 정확히 어떤 문제를 해결해주는지 체감할 수 있었습니다.
