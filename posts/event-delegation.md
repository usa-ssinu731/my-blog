목록 안의 아이템마다 클릭 이벤트가 필요할 때, 아이템 하나하나에 `addEventListener`를 붙이고 싶어질 때가 있습니다. 하지만 아이템이 동적으로 추가되거나 개수가 많아지면 이 방식은 금방 번거로워집니다. 이럴 때 쓰는 게 **이벤트 위임(event delegation)**입니다.

## 원리

이벤트는 발생한 요소에서 시작해 부모 요소로 계속 전파(버블링)됩니다. 그래서 자식 요소 각각에 리스너를 달지 않고, 공통 부모에 리스너 하나만 달아도 자식에서 일어난 이벤트를 잡아낼 수 있습니다.

```js
const list = document.querySelector('.post-list');

list.addEventListener('click', (event) => {
  const item = event.target.closest('.post-item');
  if (!item) return;

  const slug = item.dataset.slug;
  console.log('클릭한 글:', slug);
});
```

`event.target`은 실제로 클릭된 요소(예: 안에 있는 `<span>`)일 수 있으므로, `closest()`로 원하는 상위 요소까지 거슬러 올라가서 찾는 게 핵심입니다.

## 왜 유용한가

- **리스너 개수가 줄어든다**: 아이템이 100개든 1000개든 리스너는 하나면 충분해서 메모리 부담이 적습니다.
- **동적으로 추가되는 요소에도 자동으로 적용된다**: `innerHTML`로 새 아이템을 추가해도 부모에 달아둔 리스너가 그대로 작동하므로, 새 요소마다 다시 리스너를 붙일 필요가 없습니다.

이 블로그처럼 `posts/index.json`을 읽어와 글 목록을 JS로 렌더링하는 구조에서는 특히 유용합니다. 글 목록 전체를 다시 그릴 때마다 각 아이템에 리스너를 재등록할 필요 없이, 목록 컨테이너 하나에만 리스너를 걸어두면 되니까요.
